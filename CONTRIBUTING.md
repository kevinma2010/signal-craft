# Contributing to SignalCraft

SignalCraft welcomes contributions to connectors, source packs, prompts, ranking, documentation, examples, testing, and product design.

## Before a Pull Request

1. Search existing issues and pull requests.
2. Open an issue for large changes.
3. Keep changes focused.
4. Explain the user problem.
5. Add tests when behavior changes.
6. Update documentation.
7. Preserve evidence and source traceability.

## Source Proposals

Include source name, type, URL, authority, typical quality, update frequency, topics, and known limitations. Popularity alone is not sufficient.

## Prompt Proposals

Include current behavior, proposed behavior, example input, example output, expected improvement, and tradeoffs. Prompt changes must include before/after output on the fixture corpus in `fixtures/`, scored against its rubric.

## Pull Request Requirements

Provide a clear title, concise description, testing notes, sample output or screenshots when relevant, and a linked issue when applicable.

## Developer Certificate of Origin

By contributing, you certify that you have the right to submit the contribution under the project's license. A formal DCO sign-off process may be adopted as the contributor base grows.
